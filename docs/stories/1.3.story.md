# Story 1.3 — Perf: Paralelizar queries do Header e aplicar React.cache()

## Status
Ready for Review

## Story

**Como** usuário do dashboard,
**Quero** que a navegação entre páginas seja mais rápida,
**Para que** o carregamento não bloqueie por 1-2 segundos em cada troca de página.

## Acceptance Criteria

- [ ] AC1: As queries `getCurrentCompany()`, `getAllUserCompanies()` e dados do usuário são buscadas em paralelo no Header
- [ ] AC2: `getCurrentCompany()` usa `React.cache()` para evitar execuções duplicadas no mesmo request
- [ ] AC3: `getUserRole()` é incorporada na query de `getCurrentCompany()` — o papel já vem no join de `usuario_empresa`
- [ ] AC4: O Header mantém o mesmo comportamento visual e funcional de antes
- [ ] AC5: Tempo de carregamento do Header reduz de ~1.5s para ~300-400ms (estimado)

## Scope

**In scope:**
- `components/dashboard/header.tsx` — paralelizar fetches com `Promise.all()`
- `lib/supabase/queries/empresas.ts` — adicionar `React.cache()` em `getCurrentCompany()` e retornar `papel` no resultado para eliminar `getUserRole()` separado

**Out of scope:**
- Mudança de schema ou tabelas
- Outras páginas além do Header/Layout
- Implementação de cache Redis ou similar

## Root Cause

`header.tsx` executa 5-6 queries Supabase em **série** (cada uma aguarda a anterior):

```
getCurrentCompany()     → internamente: getUser() + select usuarios + select usuario_empresa
getUserRole()           → internamente: getUser() + select usuario_empresa  (REPETE getUser)
getAllUserCompanies()    → internamente: getUser() + select usuario_empresa  (REPETE getUser)
supabase.auth.getUser() → 4ª chamada a getUser() no mesmo request
select usuarios         → busca nome/avatar_url
```

Como o Header está no `layout.tsx`, esse custo é pago em **cada navegação**, bloqueando o render da página.

## Technical Context

### Correção em `lib/supabase/queries/empresas.ts`

Envolver `getCurrentCompany` com `cache()` do React para deduplicar chamadas no mesmo request:

```ts
import { cache } from "react";
import { createClient } from "@/lib/supabase/server";

export const getCurrentCompany = cache(async (): Promise<Empresa | null> => {
    // ... lógica existente inalterada
});
```

Adicionar função que retorna empresa + papel em uma única query (evita `getUserRole` separado):

```ts
export const getCurrentCompanyWithRole = cache(async (): Promise<{ empresa: Empresa | null; papel: string | null }> => {
    const supabase = await createClient();
    const { data: { user } } = await supabase.auth.getUser();
    if (!user) return { empresa: null, papel: null };

    // Query única que já traz o papel junto
    const { data: userData } = await supabase
        .from("usuarios")
        .select("empresa_atual_id")
        .eq("id", user.id)
        .single();

    if (userData?.empresa_atual_id) {
        const { data: rels } = await supabase
            .from("usuario_empresa")
            .select("papel, empresas(id, nome, logo_url, personagem_url, background_url, funil_config, segmento, ativo)")
            .eq("usuario_id", user.id)
            .eq("empresa_id", userData.empresa_atual_id)
            .limit(1);

        const rel = rels?.[0];
        if (rel?.empresas) {
            return { empresa: rel.empresas as unknown as Empresa, papel: rel.papel ?? null };
        }
    }

    // Fallback igual ao existente
    const { data: fallbackRels } = await supabase
        .from("usuario_empresa")
        .select("papel, empresas(id, nome, logo_url, personagem_url, background_url, funil_config, segmento, ativo)")
        .eq("usuario_id", user.id)
        .limit(1);

    const fallbackRel = fallbackRels?.[0];
    if (!fallbackRel?.empresas) return { empresa: null, papel: null };

    await supabase.from("usuarios").update({ empresa_atual_id: fallbackRel.empresa_id }).eq("id", user.id);
    return { empresa: fallbackRel.empresas as unknown as Empresa, papel: fallbackRel.papel ?? null };
});
```

### Correção em `components/dashboard/header.tsx`

Substituir as 5 chamadas sequenciais por `Promise.all()`:

```ts
// ANTES — sequencial (~1.5s)
const empresa = await getCurrentCompany();
const papel = empresa ? await getUserRole(empresa.id) : null;
const allCompanies = await getAllUserCompanies();
const supabase = await createClient();
const { data: { user } } = await supabase.auth.getUser();
const { data: userData } = user
    ? await supabase.from("usuarios").select("nome, avatar_url").eq("id", user.id).single()
    : { data: null };

// DEPOIS — paralelo (~300-400ms)
const supabase = await createClient();
const [
    { empresa, papel },
    allCompanies,
    { data: { user } },
] = await Promise.all([
    getCurrentCompanyWithRole(),
    getAllUserCompanies(),
    supabase.auth.getUser(),
]);

const { data: userData } = user
    ? await supabase.from("usuarios").select("nome, avatar_url").eq("id", user.id).single()
    : { data: null };
```

> Nota: `userData` (nome/avatar) ainda requer `user.id`, então aguarda o `getUser()` mas em paralelo com as outras duas.

## Dev Notes

- `React.cache()` funciona apenas em Server Components — não usar em arquivos `"use client"`
- `getAllUserCompanies()` também chama `getUser()` internamente — considerar envolver com `cache()` também para evitar duplicação
- Não alterar a interface pública de `getCurrentCompany()` para não quebrar outros consumidores
- `getUserRole()` pode ser mantida no arquivo para compatibilidade com outros lugares que a usem (ex: `vendedores.ts`)
