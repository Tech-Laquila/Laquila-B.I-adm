# Story 1.1 — Realtime: Dashboard atualiza automaticamente via Supabase

## Status
Ready for Review

## Story

**Como** usuário do dashboard,
**Quero** que os dados de vendas e métricas atualizem automaticamente quando novos registros forem inseridos no banco,
**Para que** eu não precise recarregar a página para ver o número mais recente de contratos, leads e CAC.

## Acceptance Criteria

- [ ] AC1: Quando uma nova venda é inserida na tabela `vendas`, os totais de contratos (Mês/Semana/Dia) atualizam na tela sem reload manual
- [ ] AC2: Quando um registro é inserido na tabela `leads`, os contadores de leads (Mês/Semana/Dia) atualizam em tempo real
- [ ] AC3: A subscription Realtime é inicializada ao montar o componente e removida ao desmontar (sem memory leak)
- [ ] AC4: Realtime funciona apenas para usuários autenticados (respeitar RLS existente)
- [ ] AC5: Em caso de desconexão, o sistema tenta reconectar e exibe indicador visual de status (conectado/reconectando)

## Scope

**In scope:**
- Hook `useRealtimeDashboard` ou similar em `components/` ou `hooks/`
- Conversão dos componentes que exibem totais de vendas, leads e criativos para Client Components
- Subscription em `postgres_changes` nas tabelas: `vendas`, `leads`, `facebook_ads`
- Atualização dos dados de criativos (métricas de anúncios) em tempo real via tabela `facebook_ads`
- Cleanup correto no `useEffect` / `unsubscribe`

**Out of scope:**
- Realtime para metas ou outras tabelas por ora
- Push notifications / service workers
- Mudança no schema do banco
