# Story 1.2 — Fix: Remover limite de 1000 registros nas queries do dashboard

## Status
Ready for Review

## Story

**Como** usuário do dashboard,
**Quero** que todos os cálculos de leads, contratos e custos usem o total real de registros do banco,
**Para que** os KPIs, taxas de conversão e CAC sejam precisos mesmo com volumes acima de 1000 registros no período.

## Acceptance Criteria

- [ ] AC1: A query de `leads` em `dashboard.ts` retorna todos os registros do período sem truncar em 1000
- [ ] AC2: A query de `leads` resumo (mês atual) em `dashboard.ts` retorna todos os registros sem truncar
- [ ] AC3: A query de `facebook_ads` em `dashboard.ts` retorna todos os registros sem truncar
- [ ] AC4: A query de `leads` em `vendedores.ts` retorna todos os registros sem truncar
- [ ] AC5: A query de `vendas` em `vendedores.ts` retorna todos os registros sem truncar
- [ ] AC6: A query de `facebook_ads` em `vendedores.ts` retorna todos os registros sem truncar
- [ ] AC7: Com 1200 leads no mês, o dashboard exibe 1200 (não 1000)

## Scope

**In scope:**
- Adicionar `.limit(10000)` (ou valor seguro) em todas as queries de dados agregados
- Arquivos: `lib/supabase/queries/dashboard.ts` e `lib/supabase/queries/vendedores.ts`

**Out of scope:**
- Paginação de UI
- Mudança de schema ou RLS
- Outras queries de lookup (ex: `empresas.ts` que já usa `.limit(1)` corretamente)

## Root Cause

O PostgREST (engine do Supabase) tem limite padrão de **1000 rows** por query. Sem `.limit()` explícito, queries com mais de 1000 registros são **silenciosamente truncadas**, causando KPIs incorretos.

## Technical Context

### Queries afetadas e correção

**`lib/supabase/queries/dashboard.ts`:**

```ts

  .limit(10000)  // ADICIONAR

// linha ~135 — teses list
supabase.from("leads").select("nome_conta") ...
  .limit(10000)  // ADICIONAR

// linha ~25 — facebook_ads período

  .limit(10000)  // ADICIONAR

// linha ~92 — facebook_ads resumo
let qAdsResumo = supabase.from("facebook_ads").select(...) ...

```

**`lib/supabase/queries/vendedores.ts`:**

```ts
// linha ~42 — leads
let qLeads = supabase.from("leads").select(...) ...
```

### Limite escolhido
`10000` é um valor seguro para o volume atual (1200 leads/mês). Se o volume crescer, reavaliar para paginação server-side.
