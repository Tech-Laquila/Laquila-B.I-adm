# Story 1.5 — Perf: Eliminar getDashboardData() duplicado e otimizar query de teses

## Status
Ready for Review

## Story

**Como** usuário do dashboard,
**Quero** que as páginas de Vendedores e Criativos não façam queries desnecessárias,
**Para que** o tempo de carregamento seja reduzido sem perda de dados exibidos.

## Acceptance Criteria

- [ ] AC1: A página `/dashboard/vendedores` não chama `getDashboardData()` completo — apenas busca os dados de `periodos` de forma mais eficiente
- [ ] AC2: A página `/dashboard/criativos` não chama `getDashboardData()` completo — apenas busca os dados de `periodos` de forma mais eficiente
- [ ] AC3: Os cards de resumo por período (Mês/Semana/Dia) nas páginas continuam exibindo dados corretos
- [ ] AC4: A query de `tesesList` em `getDashboardData()` é movida para uma função separada e cacheada
- [ ] AC5: Nenhuma regressão nos KPIs exibidos nas páginas de Vendedores e Criativos

## Scope

**In scope:**
- `lib/supabase/queries/dashboard.ts` — extrair `getResumoperiodos()` e `getTesesList()` como funções separadas
- `app/(app)/dashboard/vendedores/page.tsx` — substituir chamada a `getDashboardData()` pela nova função menor
- `app/(app)/dashboard/criativos/page.tsx` — idem

**Out of scope:**
- Mudança nos cálculos de KPI ou lógica de negócio
- Refatoração de `getDashboardData()` em si (mantida intacta para a página principal)
- Migração de aggregation para SQL (escopo futuro)

## Root Cause

As páginas de Vendedores e Criativos chamam `getDashboardData()` completo, mas só usam o campo `periodos` do resultado:

```ts
// vendedores/page.tsx — linha ~39-42
const [{ vendedores, ... }, { periodos }] = await Promise.all([
    getVendedoresData(dataInicio, dataFim),
    getDashboardData(dataInicio, dataFim),  // ← Busca TUDO só para usar periodos
]);

// criativos/page.tsx — linha ~41-43
const [{ criativos, conjuntos, funnel }, { periodos }] = await Promise.all([
    getCriativosData(dataInicio, dataFim, teses, ...),
    getDashboardData(dataInicio, dataFim, teses),  // ← Idem
]);
```

`getDashboardData()` executa:
- Query `facebook_ads` (até 10K linhas)
- Query `leads` (até 10K linhas)
- Query `facebook_ads` resumo (até 10K linhas)
- Query `leads` resumo (até 10K linhas)
- Query `tesesList` (até 10K linhas)

...quando as páginas só precisam dos dados de resumo para calcular `periodos`.

## Technical Context

### Nova função: `getResumoPeriodos()`

Extrair de `getDashboardData()` apenas a parte que calcula os cards Mês/Semana/Dia:

```ts
// lib/supabase/queries/dashboard.ts
import { cache } from "react";

export const getResumoPeriodos = cache(async (
    teses?: string[]
): Promise<ResumoperiodoRow[]> => {
    const supabase = await createClient();
    const empresa = await getCurrentCompany();
    const empresaId = empresa?.id ?? null;
    if (!empresaId) return [];

    const inicioMesAtual = startOfBR("month");

    let qAdsResumo = supabase
        .from("facebook_ads")
        .select("data, nome_conta, valor_usado")
        .eq("empresa_id", empresaId)
        .gte("data", inicioMesAtual);
    if (teses && teses.length > 0) qAdsResumo = qAdsResumo.in("nome_conta", teses);

    let qLeadsResumo = supabase
        .from("leads")
        .select("data_inicial, data_venda, contrato_fechado, nome_conta, fez_contato")
        .eq("empresa_id", empresaId)
        .or(`data_inicial.gte.${inicioMesAtual}T00:00:00,data_venda.gte.${inicioMesAtual}T00:00:00`);
    if (teses && teses.length > 0) qLeadsResumo = qLeadsResumo.in("nome_conta", teses);

    const [{ data: adsResumoDb }, { data: leadsResumoDb }] = await Promise.all([
        qAdsResumo.limit(QUERY_MAX_ROWS),
        qLeadsResumo.limit(QUERY_MAX_ROWS),
    ]);

    const adsResumo = (adsResumoDb ?? []) as AdMinimo[];
    const leadsResumo = (leadsResumoDb ?? []) as LeadMinimo[];

    return (["Mês", "Semana", "Dia"] as const).map((label) => {
        const unit = label === "Mês" ? "month" : label === "Semana" ? "week" : "day";
        const inicio = startOfBR(unit);
        const inicioTruncado = inicio + "T00:00:00";

        const adsP = adsResumo.filter((a) => (a.data ?? "") >= inicio);
        const leadsPPeriod = leadsResumo.filter((l) => (l.data_inicial ?? "") >= inicioTruncado);
        const contratosPPeriod = leadsResumo.filter(
            (l) => l.contrato_fechado && (l.data_venda ?? "") >= inicioTruncado
        );

        const c = adsP.reduce((s, r) => s + (r.valor_usado ?? 0), 0);
        const co = contratosPPeriod.length;
        const le = leadsPPeriod.length;
        return {
            periodo: label,
            custo: c,
            contratos: co,
            cac: co > 0 ? c / co : 0,
            leads: le,
            cpa: le > 0 ? c / le : 0,
            txConversao: le > 0 ? (co / le) * 100 : 0,
        };
    });
});
```

### Query de tesesList

A query `tesesList` no final de `getDashboardData()` pode ser extraída como função separada se necessário. Por ora, manter dentro de `getDashboardData()` já que só é chamada pela página principal de Dashboard.
